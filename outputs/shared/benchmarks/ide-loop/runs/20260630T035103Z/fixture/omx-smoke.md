# OMX Smoke

## Summary
- Fixture contains a small lane-routing module plus `unittest` coverage for parsing, grouping, risk selection, validation mode, malformed input, and a quick large-input path.
- The local unit-test loop appears verifiable from the files: `tests/test_lane_router.py` imports `lane_router.py` directly and can run as a targeted unittest module/script from this directory.

## Evidence
- `README.md` says the fixture is generated by `scripts/benchmark-ide-loop` and is intended to support a RED/GREEN loop around a risk-ordering defect.
- `lane_router.py` defines `RouteRecord`, `RouteSummary`, CSV-like parsing, route grouping, max-risk tracking, validation-mode selection, and risk-priority sorting.
- `tests/test_lane_router.py` has four focused `unittest` cases, including a performance guard for 10,000 unchanged low-risk rows.

## Gaps
- No tests were run, per the constraint to avoid broad tests.
- README claims an intentional risk-ordering bug, but this was not confirmed by execution.
