## 和 GitHub 上类似项目对比(基于训练知识,非实时核实)

**先说清定位差异:** 大多数知名框架是"**让 agent 跑起来**"的运行时;我们这个是"**让 agent 安全、可信、可协作地跑**"的治理环境。不是同一个赛道,各有强弱。

| 项目 | 它强在 | 我们强在(它没有的) | 它有而我们缺的 |
|------|--------|---------------------|----------------|
| **LangGraph** | 成熟的图式状态机运行时、可持久化、生态大 | 我们的规则继承+fail-closed 安全门+双模型协作诚实审计 | 真正的运行时引擎、流式执行、大量集成 |
| **CrewAI** | 角色/任务抽象好用、上手快、社区大 | 我们的跨层规则强制+run-record 脱敏证据链 | 成熟的 agent 角色编排、工具生态 |
| **AutoGen** | 多 agent 对话编排、微软背书、研究前沿 | 我们的环境隔离边界+提交前全平台自洽审计 | 灵活的对话式协作、可扩展性 |
| **OpenHands / aider** | 真能自动改代码跑起来的成熟 agent、大量实测 | 我们的多 agent 治理+防打架合并队列+审计诚实性 | 端到端自动开发的成熟度和实战量 |

**一句话对比结论:** 在"**治理、安全、协作诚实性**"这几项上,我们做得比多数开源框架**更认真**(尤其"自审不能造假绿"+"提交前双向审计"这种,很少框架有)。但在"**运行时成熟度、生态、实战量**"上,我们**远不如**那些千星项目——它们是打磨过的引擎,我们的运行时还是半自动雏形。

**类比:** 那些框架是"**跑得快的赛车**",我们是"**装了完整安全带、行车记录仪、和双人复核制度的车**"——但发动机还只是够用级别。

## 诚实总评

**7.3/10。** 强在治理和诚实性(这恰是多数框架的盲区),弱在运行时成熟度和生态。它现在最适合的定位不是"取代 LangGraph 那种框架",而是"**在其之上/之外,提供一个可信协作与治理层**"。要真正对标那些项目,得把编排从"半自动"推到"能自己跑的运行时",而这正是被诚实标为 4 分的那块。

**给用户的实话:** 别拿它当"比 LangGraph 强的框架"去对外宣称——不实。它真正的独特价值是"**两个 AI 能可信地、留痕地、不打架地协作开发**",这一点是稀缺的。
